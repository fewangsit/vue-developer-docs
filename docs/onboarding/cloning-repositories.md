# Cloning Repositories

There are a couple of repositories that you should clone:

* [**Wangsit Micro-Frontend Boilerplate App**](https://github.com/fewangsit/wangsit-mfe-boilerplate-app)
*   [**Wangsit VueJS Component Library**](https://github.com/fewangsit/wangsvue)

    Clone this repository to browse the playground folder, where you'll find example implementations of each Vue component. This will be helpful in the later Slicing step. Additionally, you can run the repository to test each component.

Here’s how to clone the repositories:

1. Click on the links above to go to each of the repositories’ pages.
2.  For the Boilerplate repository, on the top right of the page, click `Use this template`, and choose `Create a new repository`. Note that you need to create it using the GitHub account given to you by the company, not your personal GitHub account.

    <img src="https://cdn.hashnode.com/res/hashnode/image/upload/v1734074475215/7607c91b-8c2d-4981-b963-5b53e02c8481.png?auto=compress,format&#x26;format=webp?auto=compress,format&#x26;format=webp&#x26;q=75" alt="" data-size="original">
3.  Put in your nickname as the repository name, and make the repository private. Create the repository.

    ![](https://cdn.hashnode.com/res/hashnode/image/upload/v1734075073325/49e97231-0b6f-4e0c-b6f1-89ce0b2a58db.png?auto=compress,format\&format=webp?auto=compress,format\&format=webp\&q=75)
4.  Once the repository is created, click the `<> Code` button and copy the URL under HTTPS.

    ![](https://cdn.hashnode.com/res/hashnode/image/upload/v1734075141463/c7309d45-1675-47f0-87e5-b6204224a1e9.png?auto=compress,format\&format=webp?auto=compress,format\&format=webp\&q=75)
5. Open VS Code, and from the toolbar, click `File > Open Folder` to open the folder where you want to clone the repository.
6. Open the terminal with `` Ctrl + ` `` (backtick, the key above your `Tab` key).
7. Type `git clone`, then press `Spacebar` and paste the URL you copied in step 4 into the terminal.
8. Press `Enter`, and wait for the repository to be cloned.
9. Repeat steps 4 to 8 for the Component Library repository.
